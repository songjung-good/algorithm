# clone 오류

## 상황
  - 나의 github에는 algorithm이라는 repo가 존재한다.
  - 해당 repo에 수정을 하기위해 매번 하던 방법과 같이
    github notebook을 이용하여 폴더에 clone 했다.
  - 오류 발생 후 clone 중단

## 원인 1
  - 다른 이름의 레포에 algorithm이라는 폴더가 있었다
  - 해당 폴더를 먼저 같은 위치에 받았다.
  - 그 폴더를 밀어버리고 같은 위치에 clone 하는 것이 문제가 아니었을까?

## 방법
  - 기존 폴더 지우고 다시 클론하기
  - rm -rf 를 이용해서 지우기

## 원인(진짜)
  - 파일중 하나의 제목에':'가 포함되었다.
  - 허용되지 않는 폴더명이라 문제가 발생했다.

## 방법
  - github에 들어가서 강제로 폴더명 수정
